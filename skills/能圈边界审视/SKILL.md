# 能圈边界审视

> **核心理念**：芒格说"知道自己能圈边界的人，比能圈很大但不知道边界的人更安全。"界定边界，守住核心。

## 使用场景

- 有人找你合作/外包
- 你在考虑接一个新的需求
- 你想尝试一个新的业务方向
- 你在评估一个项目是否适合自己
- 你在纠结"该不该接这个活"

---

## 核心概念：能圈（Circle of Competence）

```
        ┌─────────────────────────────────────┐
        │           你不知道的                  │
        │         (不要碰的领域)                 │
        │    ┌─────────────────────────┐       │
        │    │      边界模糊区          │       │
        │    │   (谨慎尝试/找合作)       │       │
        │    │   ┌───────────────┐     │       │
        │    │   │   核心能圈     │     │       │
        │    │   │  (你的强项)    │     │       │
        │    │   │  已验证的能力  │     │       │
        │    │   └───────────────┘     │       │
        │    └─────────────────────────┘       │
        └─────────────────────────────────────┘
```

## 执行流程

### Step 1: 梳理你的核心能圈

首先，帮助用户梳理已有的能力：

| 能力类型 | 具体内容 | 验证证据 |
|---------|---------|---------|
| **技术能力** | 你最擅长的技术栈/工具 | 已完成的项目、用户反馈 |
| **业务能力** | 你最熟悉的业务领域 | 行业经验、客户积累 |
| **资源能力** | 你拥有的独特资源 | 人脉、渠道、数据 |
| **学习能力** | 你能快速上手的方向 | 过往学习曲线 |

### 验证标准

- **✅ 已验证**：有真实用户付费、有成功案例、有持续收入
- **⚠️ 待验证**：有能力但未商业化、有想法但未执行
- **🚫 未验证**：只是觉得自己能行、没有实际证据

---

## Step 2: 分析新需求/新方向

针对用户提出的新需求，进行拆解：

### 需求拆解表

| 需求模块 | 具体内容 | 能力匹配度 |
|---------|---------|-----------|
| 模块1 | ... | ✅ 核心能力 / ⚠️ 边界能力 / ❌ 圈外能力 |
| 模块2 | ... | ... |
| 模块3 | ... | ... |
| 模块4 | ... | ... |

### 能力匹配度分析

对每个模块进行详细分析：

#### ✅ 在核心能圈内

```
能力：[具体能力]
匹配理由：...
已有验证：...
投入成本：低
风险等级：🟢 低
```

#### ⚠️ 在边界模糊区

```
能力差距：[缺少什么]
已有基础：[有什么]
补充成本：[需要投入什么]
风险评估：...
建议：学习/合作/外包
```

#### ❌ 在能圈之外

```
缺失能力：[具体是什么]
学习成本：[时间/精力/金钱]
机会成本：[如果学习这个，会放弃什么]
风险等级：🔴 高
建议：拒绝/找合作伙伴
```

---

## Step 3: 能圈匹配度评分

### 计算公式

```
能圈匹配度 = (核心能力模块数 × 3 + 边界能力模块数 × 1) / (总模块数 × 3) × 100%
```

### 评分解读

| 匹配度 | 评价 | 建议 |
|-------|------|------|
| **80%+** | 🟢 高度匹配 | 可以直接推进 |
| **50-80%** | 🟡 部分匹配 | 需要补充能力或找合作 |
| **50%以下** | 🔴 低度匹配 | 建议拒绝或大幅调整 |

---

## Step 4: 决策建议

### 情况一：高度匹配（80%+）

```
✅ 建议：可以推进

优势：
- ...
- ...

注意事项：
- 边界模块可能需要关注：...
- 建议预留学习/适应时间：X天/周
```

### 情况二：部分匹配（50-80%）

```
⚠️ 建议：谨慎推进，需要补充

需要补充的能力：
| 能力 | 补充方式 | 预计成本 | 优先级 |
|-----|---------|---------|-------|
| ... | 学习/合作/外包 | ... | P0/P1/P2 |

两种策略：
1. **能力补充策略**：先学习/准备，再推进
   - 预计时间：...
   - 预计投入：...

2. **合作策略**：找有这个能力的人合作
   - 合作方式：...
   - 利益分配建议：...
```

### 情况三：低度匹配（50%以下）

```
❌ 建议：不建议直接推进

核心原因：
- ...
- ...

替代方案：
1. **专注核心**：把精力放在匹配度更高的方向
   - 当前最值得做的：...

2. **合作模式**：如果确实想做，找合适的合作伙伴
   - 需要找什么样的合作伙伴：...

3. **外包模式**：只做自己擅长的部分
   - 你负责：...
   - 外包：...
```

---

## Step 5: 能圈扩展建议

如果用户决定要扩展能圈：

### 能圈扩展原则

1. **渐进式扩展**：每次只扩展一小步
2. **验证后扩展**：先小规模验证，再大举投入
3. **保留核心**：扩展不能影响核心业务的稳定性

### 扩展路径规划

```
当前能圈 ──→ 第一阶段扩展 ──→ 第二阶段扩展 ──→ 目标能圈
   │              │                │              │
   │              │                │              │
   ▼              ▼                ▼              ▼
已验证        小规模验证        中等规模        完整扩展
```

### 扩展检查清单

- [ ] 核心业务不受影响
- [ ] 有足够的时间和精力
- [ ] 有明确的学习/验证路径
- [ ] 有退出的安全边际
- [ ] 不是被热点/诱惑驱动

## 注意事项

1. **诚实面对边界**：不要因为想赚钱/想尝试就欺骗自己"这在我的能力范围内"
2. **能圈是动态的**：可以扩展，但要渐进式、有验证
3. **专注核心价值**：能圈外的机会，可能是别人的核心能圈，不如合作
4. **考虑机会成本**：扩展能圈的时间，是否应该用来强化核心能圈？
